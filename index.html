HTML

  • The head element contains information about the webpage
  • Only the title from the head element is visible in the Browser tab title
  • The body element contain the information for the user
  • A margin indicates how much space we want around the outside of the element
  • A padding indicates how much space we want around the content inside the element
  • Don't forget to read every day something new about how to improve the HTML

CSS

  • CSS is useful to give the "touch" of persnalization to a HTML
  • The most important : you need to link the script.scc inside the "head" of the HTML
  • You can use styling for multiple elements at once by using comma inbetween the elements name
  • You can use "id" and "classes" to style specific elements

Git

  • git status : check the current branc where we are
  • git checkout -b branch-name : create a new branch and switch to it
  • git pull origin main : pull the last version of main from gitHub
  • Order of git commands after finishing editing : staging(add), commiting(commit), checking version(checkout), pushing(push)

JavaScript

  • A variable is a named container that allows us to store data in our code,
  • Control flow is the order in which a computer executes code in a script
  • Using function can help with reducing the amount of repetitive coding
  • dont't forget that the link toward the JS script file should be at the end of the html file, just before the ending tag of the "body" element
  • Arrays are very useful for working with multiple values
  • The google dev tool give acces to the console inside the webpage and open a quick acces to experiment with the JS code
